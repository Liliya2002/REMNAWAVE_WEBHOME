# Конфиги RemnaWave (Xray)

Этот раздел — про настройку Xray-конфигурации в панели RemnaWave: какие протоколы рекомендованы, как создать конфиг, как добавить inbound и подключить ноду.

## Поддерживаемые протоколы

| Протокол | Где использовать | Плюсы | Минусы |
|---|---|---|---|
| **VLESS + Reality** | основной выбор | хорошо обходит DPI, не палится TLS-handshake | требует "донорский" SNI |
| **VLESS + WS + TLS (CDN)** | страны с глубокой блокировкой | можно прятаться за Cloudflare/иной CDN | нужен домен и сертификат |
| **Trojan** | резервный | простой, маскируется под HTTPS | детектится продвинутым DPI |
| **Shadowsocks 2022** | для совместимости | просто, быстро | без обфускации — палится |

> **Рекомендация:** на каждой ноде включай **VLESS + Reality** как основной + **Trojan** как fallback.

## Создание Xray-конфига

1. Зайди в RemnaWave → **Configs** → **Create config**.
2. Шаблон — `vless-reality-base` (если есть в твоей панели). Иначе — пустой.
3. Заполни обязательные поля:
   - `tag` — уникальный, например `vless-reality-de1`
   - `port` — рекомендуется **443** (или другой стандартный TLS-порт)
   - `dest` — донорский SNI, например `www.microsoft.com:443`
   - `serverNames` — список SNI, например `["www.microsoft.com"]`
   - `shortIds` — оставить дефолт или сгенерировать
   - `privateKey` / `publicKey` — сгенерируются автоматически

4. Нажми **Save** → конфиг готов к назначению на ноды.

## Пример минимального VLESS+Reality inbound

```json
{
  "tag": "vless-reality-de1",
  "listen": "0.0.0.0",
  "port": 443,
  "protocol": "vless",
  "settings": {
    "clients": [],
    "decryption": "none"
  },
  "streamSettings": {
    "network": "tcp",
    "security": "reality",
    "realitySettings": {
      "show": false,
      "dest": "www.microsoft.com:443",
      "xver": 0,
      "serverNames": ["www.microsoft.com"],
      "privateKey": "<заполнится автоматически>",
      "shortIds": [""]
    }
  },
  "sniffing": { "enabled": true, "destOverride": ["http", "tls"] }
}
```

## Назначение конфига на ноду

1. RemnaWave → **Nodes** → выбрать ноду.
2. Вкладка **Inbounds** → **Assign** → выбрать только что созданный конфиг.
3. Нажать **Sync** (или дождаться авто-синхронизации в течение ~30 сек).
4. Проверить статус ноды — должен стать **Online** с зелёной точкой.

## Чек-лист после создания

- [ ] Inbound отображается в RemnaWave с **подсчитанными клиентами**
- [ ] На ноде через `ss -tlnp` виден слушающий порт
- [ ] Тестовый пользователь успешно подключается
- [ ] В логах Xray на ноде нет ошибок про сертификаты / SNI
- [ ] Скорость подключения — выше 50 Мбит/с (если нет — проверь `dest` SNI)

## Частые ошибки

| Симптом | Причина | Решение |
|---|---|---|
| Клиент подключается, но трафик не идёт | Не пробрасывается на роутер | Проверь `outbounds` в конфиге Xray |
| Reality не работает | Невалидный `dest` | Проверь, что `curl https://<dest>` отвечает 200/301 |
| Связь рвётся через 10–20 сек | DPI режет по SNI | Замени `serverNames` на менее палевный сайт |
